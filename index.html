Getting started

Setup

Get Claude running in the right environment from day one. The choices you make here affect how much you can do later.

Shortcut: auto-setup with the bootstrap prompt

Rather than doing all of this manually, paste the bootstrap prompt into a fresh Claude desktop chat and it will handle steps 2–4 for you automatically — creating your vault structure, skill files, and saving your preferences to memory.

→ Get the bootstrap prompt (claude-bootstrap.md)
1

Create an account

Go to claude.ai and sign up. The free plan gives you access to Sonnet. A Pro plan unlocks Opus, higher limits, and the desktop app.

→ claude.ai
2

Download the desktop app (recommended)

Mac and Windows. The desktop app unlocks local file access, connectors, and co-work — things the web app simply can't do.

→ claude.ai/download
3

Set up a notes system — Obsidian

Obsidian (free) is a local markdown vault. Ask Claude to save project notes as .md files in your vault. This keeps your work AI-platform agnostic — readable anywhere, not locked to Claude.

→ obsidian.md

Once installed, tell Claude: "Save a memory that my Obsidian vault is at ~/Documents/Vault — use this for all project notes."

4

Optional: voice input

TypeWhisper runs locally and converts speech to text. Useful for longer, more detailed prompts without typing.

→ typewhisper.app

Getting started

Web app vs desktop app

Both work. But they're not equal.

Web app

  • Works on any device, no install
  • Great for quick questions
  • Shareable conversation links
  • No local file access
  • No MCP connectors
  • No co-work support

Desktop app ★ recommended

  • Access to local files and folders
  • MCP connectors (Gmail, Drive, etc.)
  • Co-work for file and task management
  • Local skill files for reuse
  • Better for ongoing projects
  • Requires installation
  • Pro plan for full features

Short version: Use the web app to start. Switch to the desktop app as soon as you're doing real project work.

Choosing a model

Three models. Getting this right affects both quality and cost — significantly.

Claude Opus
Use sparingly		 Complex planning, strategy, high-stakes decisions only Slower Highest
Claude Sonnet
★ Your workhorse		 Research, writing, coding, most everyday tasks Fast Balanced

Lightweight tasks		 Email replies, summaries, anything low-effort Fastest Lowest

Using Claude

Chat and planning

Chat is Claude's default mode. Used well, it's the best planning partner you've had.

Map out the problem

Describe what you want to build. Ask Claude what assumptions you're making and what you might be missing.

Identify the market

Competitors, differentiation, technology used — give Claude a clear scope and it'll surface what matters.

Break into phases

Ask Claude to break a big project into phases with clear deliverables and dependencies.

Stress-test the plan

Ask: "What are the three most likely reasons this fails?" Worth doing before you commit to a direction.

"I want to build an app for guitar tuners. Research the existing tools in this space, identify how they work, find the top competitors, and help me understand where I could differentiate. Then help me plan Phase 1."

Projects — persistent context

Claude's built-in Projects feature groups related conversations and gives Claude persistent context. Set this up for any ongoing work so you're not re-explaining everything every session.

Research workflow

Always research before building. Even if you think you know enough.

Research

1

Give Claude a clear scope

Don't say "research X." Say: "Research the top 5 competitors in [space], what technology they use, their pricing model, and where they're weak."

2

Feed it sources yourself

Claude is better at analysis than discovery. If you've found a useful article or competitor site, paste the URL in. Don't rely on Claude to find everything.

3

Save output as a markdown file

Ask Claude to save research to your Obsidian vault. Searchable later, referenceable in future chats, and it reduces token waste downstream.

4

Enable web search

Toggle it on in Claude.ai for research tasks. Claude's training has a cutoff — web search gives it live sources.

Watch out for: Claude hallucinating competitor details, pricing, or sources. Always verify specific claims before acting on them.

Using Claude

File and project setup

Claude has three modes for working with files. Knowing which to use makes the difference between fast and frustrating.

Co-work — the right tool for file setup

Co-work (desktop app only) gives Claude direct access to your local files and folders. This is what you want for creating project structures, writing and editing documents, and generating spreadsheets. It works on your machine, not in the chat window.

"I'm starting a freelance design studio. Create a folder structure at ~/Documents/Studio for client projects, proposals, assets, and invoices — then generate a client tracker spreadsheet in the root."

Folder structures

Describe your business or project type. Co-work creates the folders, subfolders, and starter files — ready to use immediately.

Spreadsheets and trackers

Ask for expense trackers, project timelines, or client logs as .xlsx files. Co-work writes them directly to your machine.

Markdown notes and briefs

Project briefs, research summaries, meeting notes — saved as .md files in your Obsidian vault, readable in any tool.

Replicating structures

Set up a structure once for one client or project, then ask Co-work to copy and rename it for every new engagement.

Chat — planning before you build

Before touching files, use regular Claude chat to figure out what you actually need. Chat is fast and cheap — use it to think through the structure before Co-work builds it.

"I'm setting up a solo consulting business. What folder structure would you recommend for managing clients, proposals, contracts, and invoices? I'll use Obsidian for notes."

Claude Code — for developers

If you're working in a codebase, Claude Code is the right tool. It reads your entire project, scaffolds new features, creates config files, and sets up environments — all from the terminal. See the Claude Code section for setup.

Platform-agnostic notes: Always save project notes as .md files. They work in any tool — Obsidian, Notion, VS Code — and you're never locked to one platform.

Power features

Token management

This is the section most people skip — and it's the one that costs them the most. Claude is geared to use tokens, and you need to be strict about how and when it does.

How tokens actually work

Every message uses tokens (roughly one word = one token). What most people miss: every single prompt includes your entire conversation history.

Token cost per message as a conversation grows

Message 1 — just your prompt
Message 3 — prompt + 2 full exchanges resent
Message 5 — now burning credits fast
Message 10+ — most of each message is dead history

After 4–5 replies in a single chat, you're spending most of each message just resending history. Long threads don't make Claude smarter — they make it slower and more expensive.

How to fix it

1

Start new chats often

A new chat doesn't mean starting from scratch — it means not carrying dead weight. One topic per chat.

2

Use your MD files as context

Keep a well-maintained brief.md per project. At the start of a new chat: "Read my project brief at ~/Vault/ProjectX/brief.md before we start." Claude gets full context instantly. This is how you start fresh without starting over.

3

Keep documents lean

Don't paste entire codebases or long docs — summarise first and let Claude ask if it needs more.

4

Use skills instead of repeating yourself

If you're explaining your brand voice or project requirements in every chat — that's wasted tokens. Write it once as a skill file and reference it.

5

Pick the right model

Haiku costs a fraction of Opus. Burning Opus tokens on email summaries or low-stakes tasks is the fastest way to hit your limits.

6

Developers: use RTK (Rust Token Killer)

Specifically built to significantly reduce tokens used in Claude Code sessions. If you're running Claude Code regularly, it's a meaningful saving.

→ RTK on GitHub

The mindset shift: Starting a new chat isn't starting over. If your markdown files are up to date, Claude picks up exactly where you left off — with a clean context window and no wasted history.

Skills and repositories

Skills are reusable instruction files that tell Claude how to behave for specific tasks. Don't build them all from scratch — there are public repositories with thousands already made.

Finding skills

skills.sh — the open agent skills directory

A leaderboard of publicly available skills with install counts, GitHub repo links, and compatibility info. Browse by category or sort by stars to find what's well-used and trusted.

→ skills.sh

MCP Market — skills for Claude

Another directory specifically focused on Claude-compatible skills and MCP integrations. Good for finding skills that work with specific tools and workflows.

→ mcpmarket.com/tools/skills

Awesome MCP Servers

Community-curated list of MCP server configs covering databases, APIs, and local tools. The go-to starting point for MCP setup.

→ github.com/punkpeye/awesome-mcp-servers

Anthropic official skills

Anthropic's own skills repository on GitHub — these are the reference implementations and include skills for Claude Code, document creation, design, and more.

→ github.com/modelcontextprotocol → Claude connectors directory

How to install a skill

1

Browse and find one. Go to skills.sh or MCPMarket. Check the star count on its GitHub repo — high stars means lots of people have used and validated it.

2

Read what it does. Look at the skill's README before installing. Make sure it actually fits your use case — don't just grab the most popular one.

3

Install option A — give Claude the GitHub URL. Tell Claude: "Download and install the skill from github.com/owner/repo" and it will fetch and set it up for you.

4

Install option B — download the zip yourself. Download the repo as a zip from GitHub and attach it to a Claude chat. Tell Claude to extract and install it into your skills folder.

5

Test it before relying on it. Use an example prompt relevant to your actual work. Does it behave the way you expected? If not, adjust or try a different skill.

Star count is your quality signal. On GitHub and skills.sh, more stars = more people have used it and found it valuable. Don't install skills with very few stars without reading them carefully first.

Power features

MCPs and connectors

MCPs let Claude connect directly to your tools. Instead of copy-pasting between apps, Claude reads from and writes to them directly. This is where the desktop app gets genuinely powerful.

Installing connectors

1

Open Settings in the desktop app

Go to Settings → Connectors. You'll see all available integrations with one-click setup.

2

Connect your most-used tools first

Gmail and Google Drive are useful for almost everyone. Add more as you find the need.

3

Use them naturally in conversation

Once connected, just reference the tool: "Check my Gmail for anything urgent this week and give me a summary."

Most useful connectors

✉

Gmail

Claude reads your inbox, summarises what's happening, drafts replies, flags important threads, and writes outgoing emails. Ask for a morning briefing and it'll surface what actually needs your attention — great for high-volume inboxes.

📁

Google Drive

Claude sees all your Drive files — reads docs, summarises reports, references old proposals, and creates new files directly. No exporting, no copy-pasting. Especially useful when starting a new client project and you want Claude to read existing work for context.

📅

Google Calendar

Check upcoming meetings, draft prep notes, suggest scheduling blocks, and plan your week around existing commitments.

✎

Notion / Linear

Read and create tasks, update project docs, pull in context from your workspace — Claude becomes a genuine project management assistant.

📊

HubSpot

Connect Claude to your CRM and ask it to read contact histories, analyse pipeline health, identify follow-up gaps, and diagnose why deals are stalling. Sales teams can get analysis without logging into HubSpot and running manual reports.

🔗

GitHub

Review PRs, search issues, read repo contents — without leaving Claude. Pairs well with Claude Code for developers.

Advanced: You can run local MCP servers to connect Claude to tools without official connectors. See the skills repositories section for community-built options covering dozens of additional services.

Power features

Claude Code

A command-line tool that gives Claude direct access to your codebase. It reads files, writes code, runs commands, and iterates — all from your terminal.

What Claude Code can do

Read your entire codebase for context, write and edit files directly, run terminal commands, debug errors, refactor code, write tests, and explain every step.

What it's not for

Quick one-off questions or tasks that don't involve a codebase. For those, use regular Claude chat with Sonnet.

1

Install via terminal

Run npm install -g @anthropic-ai/claude-code. Requires Node.js.

2

Navigate to your project

Open a terminal in your project folder and run claude. Claude Code reads your project structure automatically.

3

Give it a task

"Add input validation to the login form" or "Why is this function returning undefined?"

Good use cases: building features, refactoring, writing tests, migrating frameworks, debugging tricky errors, spinning up new projects from scratch.

Token note: Claude Code burns through tokens quickly. Use RTK (Rust Token Killer) to significantly reduce usage. Keep git commits small so you can revert if something goes wrong.

Wrapping up

Tips and habits

The things that separate people who get average results from those who get genuinely good ones.

1

Always research before building

Even if you think you know enough. The research step surfaces blind spots and gives Claude context for better output downstream.

2

Give Claude references, not just instructions

If you want a website redesigned, show it examples. If you want copy in a certain tone, paste a sample. Claude works better with models to reference than abstract descriptions alone — especially for design and style work.

3

Push back when something's not right

Claude isn't always correct on the first try. Be specific: "The tone is too formal — rewrite this like you're talking to a friend."

4

Build skills for recurring work

Anything you explain more than twice should be a skill file. Write it once, reference it forever.

5

Keep conversations scoped

One topic per chat. Mixing unrelated projects in one thread wastes tokens on irrelevant history and dilutes Claude's focus.

6

Treat Sonnet as your default

Reach for Opus only when you genuinely need deep reasoning on a complex decision. Everything else — Sonnet.

7

Maintain your markdown files

An up-to-date brief.md means any new chat can get full context instantly. This is the whole system working as intended.

8

Use memory intentionally

Tell Claude the things you want it to remember: your role, your vault path, your output preferences. Saves time and tokens across every session.

You're set. Work through the guide, start with the basics, and layer in MCPs, skills, and Claude Code when you're ready. The best way to learn Claude is to use it daily.
